Địa chỉ IPv4 & Kỹ thuật chia mạng con (Subnetting)

Đây là chương quan trọng nhất về tính toán trong môn học. Học xong chương này bạn sẽ tính được: từ 1 địa chỉ IP + prefix length (/24, /26...) → suy ra Network ID, Broadcast, dải host khả dụng, Subnet Mask.

4.1. Địa chỉ IP là gì?

Trong mạng TCP/IP, có 3 cách để xác định một máy tính:

  • Địa chỉ vật lý (MAC): Ghi sẵn trên NIC, 48 bit, không thay đổi. Dùng trong LAN.
  • Địa chỉ IP: Logic, 32 bit (IPv4) hoặc 128 bit (IPv6), có thể thay đổi. Dùng trong liên mạng.
  • Tên miền: Dễ nhớ (google.com). DNS chuyển sang IP.
🏠 Ví dụ đời thường

MAC = số CMND (không đổi theo thời gian, định danh vĩnh viễn). IP = địa chỉ nhà (có thể đổi khi chuyển nhà). Tên miền = tên người (dễ nhớ, dùng để tìm trong danh bạ → ra số CMND và địa chỉ).

4.2. Cấu trúc địa chỉ IPv4

Địa chỉ IPv4 dài 32 bit, được chia thành 4 octet (mỗi octet 8 bit), biểu diễn dạng thập phân (dotted-decimal), cách nhau bởi dấu chấm. 

Ví dụ: 192.168.1.2

Nhị phân: 11000000 . 10101000 . 00000001 . 00000010
              192    .    168    .      1    .      2

Mỗi octet: 8 bit → giá trị thập phân 0 - 255

Địa chỉ IP gồm 2 phần: Network ID (phần mạng, các bit cực trái) và Host ID (phần máy, các bit cực phải). Ranh giới giữa 2 phần được xác định bởi Subnet Mask.

4.3. Phân lớp địa chỉ IP

IPv4 chia thành 5 lớp: A, B, C (dùng được), D (multicast), E (dự trữ).

⚠️ Cần nhớ
  • 127.x.x.x là Loopback (Localhost) — không gán cho máy, dùng để kiểm tra chính máy.
  • 127.0.0.1 = địa chỉ loopback mặc định.
  • Lớp A: 126 mạng (không phải 128) vì mất mạng 0.0.0.0 và 127.0.0.0 (loopback).
  • Lớp C phổ biến nhất trong mạng gia đình/công ty nhỏ (254 host đủ dùng).

4.4. Dải IP Private (IP nội bộ)

IP Private là không gian địa chỉ dùng riêng trong tổ chức, không được định tuyến ra Internet công cộng. Nhà cung cấp Internet sẽ chặn các gói tin có IP Private. Trong mạng nội bộ, dùng IP Private + NAT để ra Internet.

LớpDải IP PrivateSố lượng

B172.16.0.0 → 172.31.255.2551.048.576
C192.168.0.0 → 192.168.255.25565.536
🏠 Ví dụ đời thường

Wifi nhà bạn thường cấp IP kiểu 192.168.1.x — đó là IP Private lớp C. Khi bạn vào web, router dùng NAT (Network Address Translation) để "dịch" IP nội bộ thành IP public của nhà mạng → Internet chỉ thấy IP của router, không thấy IP nội bộ của bạn.

4.4.1. Dải APIPA

Khi máy không nhận được IP từ DHCP Server (sau 60 giây), Windows tự gán IP thuộc dải APIPA (Automatic Private IP Addressing): 169.254.x.x (đây là dấu hiệu mạng có vấn đề).

4.5. Subnet Mask (Mặt nạ mạng)

Subnet Mask là số 32 bit dùng để xác định trong 1 địa chỉ IP, phần nào là Network, phần nào là Host.

  • Các bit tương ứng phần Network = 1
  • Các bit tương ứng phần Host = 0

Subnet Mask mặc định theo lớp:

  • Lớp A: 255.0.0.0
  • Lớp B: 255.255.0.0
  • Lớp C: 255.255.255.0
Ví dụ tính Net ID từ IP + Mask

Cho IP: 194.200.101.101, Mask: 255.255.255.0

IP   : 194 . 200 . 101 . 101
Mask : 255 . 255 . 255 .   0
                    ↑
        Octet 4 là phần Host (vì mask octet 4 = 0)

→ Network ID: 194 . 200 . 101 .   0   (bit Host = 0)
→ Host ID   :   0 .   0 .   0 . 101

4.6. Ký pháp CIDR (/n)

CIDR (Classless Inter-Domain Routing) là cách viết tắt Subnet Mask bằng số bit 1 liên tiếp.

  • /24 = 24 bit Network = 255.255.255.0 (8 bit host)
  • /16 = 16 bit Network = 255.255.0.0 (16 bit host)
  • /8 = 8 bit Network = 255.0.0.0 (24 bit host)
  • /26 = 26 bit Network = 255.255.255.192 (6 bit host)
  • /30 = 30 bit Network = 255.255.255.252 (2 bit host — chỉ 2 IP khả dụng, dùng cho point-to-point)

4.7. 3 loại địa chỉ trong 1 Subnet

Trong một dải mạng, có 3 loại địa chỉ đặc biệt cần phân biệt:

  1. Địa chỉ Network (Network ID): Tất cả bit phần Host = 0. Định danh cả dải mạng, KHÔNG gán cho thiết bị.
  2. Địa chỉ Broadcast: Tất cả bit phần Host = 1. Gửi tới tất cả máy trong mạng, KHÔNG gán cho thiết bị.
  3. Địa chỉ Host: Ít nhất 1 bit phần Host = 1 (nhưng không phải tất cả = 1). Địa chỉ khả dụng, gán được cho PC/Router.
💡 Cách xác định loại địa chỉ

Nhìn vào các bit host (bit cực phải, số bit = 32 - prefix):

  • Tất cả = 0 → địa chỉ Network
  • Tất cả = 1 → địa chỉ Broadcast
  • Có ít nhất 1 bit = 1 và ít nhất 1 bit = 0 → địa chỉ Host

4.8. Subnetting (Kỹ thuật chia mạng con)

4.8.1. Tại sao phải chia mạng con?

  • Giảm kích thước, phù hợp mô hình công ty
  • Phân cấp quản lý
  • Giảm nghẽn mạng (giới hạn phạm vi broadcast)
  • Cô lập sự cố trong từng subnet
  • Tăng bảo mật (áp dụng policy khác nhau từng subnet)
  • Áp dụng cấu hình khác nhau cho từng subnet

4.8.2. Nguyên tắc

Lấy (mượn) một số bit ở đầu phần Host ID để chuyển thành phần Network ID. Số bit mượn tối thiểu = 2, tối đa:

  • Lớp A: tối đa 22 bit → 2^22 - 2 = 4.194.302 subnet
  • Lớp B: tối đa 14 bit → 2^14 - 2 = 16.382 subnet
  • Lớp C: tối đa 6 bit → 2^6 - 2 = 62 subnet

4.9. Các công thức tính cốt lõi

📐 Công thức

Gọi n = số bit mượn (để chia subnet), m = số bit Host còn lại.

  • Số Subnet mới = 2^n
  • Số Host khả dụng/subnet = 2^m - 2 (trừ Network và Broadcast)
  • Bước nhảy (Block Size) = 256 - giá trị thập phân của Octet mượn cuối cùng

Ví dụ: /26 (lớp C, mượn 2 bit, host còn 6 bit)

  • Số subnet = 2^2 = 4
  • Số host/subnet = 2^6 - 2 = 62
  • Block size = 256 - 192 = 64 → các Net ID kế tiếp: .0, .64, .128, .192

4.10. Ví dụ minh hoạ chi tiết

📌 Ví dụ 1: IP 192.168.1.2/24

Bước 1: SM 24 bit → 8 bit host → host rơi vào octet 4.

Bước 2: Octet 4 = 2 = 00000010 → có bit ≠ 0 → địa chỉ Host.

Bước 3: Tính Network: bit host = 0 → octet 4 = 00000000 → 192.168.1.0/24

Bước 4: Tính Broadcast: bit host = 1 → octet 4 = 11111111 = 255 → 192.168.1.255/24

Bước 5: Dải khả dụng: 192.168.1.1 → 192.168.1.254 (254 host)

📌 Ví dụ 2: IP 184.156.12.165/27

Bước 1: SM 27 bit → 5 bit host (cực phải, trong octet 4).

Bước 2: Octet 4 = 165 = 10100101. Bit host là 5 bit phải: 00101 ≠ 0 → địa chỉ Host.

Bước 3: Tính Network: 5 bit host = 0 → octet 4 = 10100000 = 128+32 = 160 → 184.156.12.160/27

Bước 4: Tính Broadcast: 5 bit host = 1 → octet 4 = 10111111 = 191 → 184.156.12.191/27

Bước 5: Dải khả dụng: 184.156.12.161 → 184.156.12.190 (30 host)

📌 Ví dụ 3: IP 184.156.168.30/19

Bước 1: SM 19 bit → 13 bit host (cực phải, ở octet 3 và 4).

Bước 2: Octet 3 = 168 = 10101000. Bit host ở octet 3 là 5 bit phải: 01000 ≠ 0 → địa chỉ Host.

Bước 3: SM dạng nhị phân: 11111111.11111111.11100000.00000000 → 255.255.224.0

Bước 4: Network: bit host = 0 → octet 3 = 10100000 = 160, octet 4 = 0 → 184.156.160.0/19

Bước 5: Broadcast: octet 3 = 10111111 = 191, octet 4 = 255 → 184.156.191.255/19

Bước 6: Dải khả dụng: 184.156.160.1 → 184.156.191.254 (8190 host)

📌 Ví dụ 4: IP 175.25.37.110/28

Bước 1: SM 28 bit → 4 bit host (octet 4).

Bước 2: Octet 4 = 110 = 01101110. Bit host là 4 bit phải: 1110 ≠ 0 → địa chỉ Host.

Bước 3: Network: 4 bit host = 0 → octet 4 = 01100000 = 96 → 175.25.37.96/28

Bước 4: Broadcast: octet 4 = 01101111 = 111 → 175.25.37.111/28

Bước 5: Dải khả dụng: 175.25.37.97 → 175.25.37.110 (14 host). SM = 255.255.255.240

📌 Ví dụ 5: IP 163.128.170.0/19

Bước 1: SM 19 bit → 13 bit host (octet 3, 4).

Bước 2: Octet 3 = 170 = 10101010. Bit host ở octet 3 là 5 bit phải: 01010 ≠ 0 → địa chỉ Host.

Bước 3: SM: 255.255.224.0

Bước 4: Network: octet 3 = 10100000 = 160 → 163.128.160.0/19

Bước 5: Broadcast: octet 3 = 10111111 = 191, octet 4 = 255 → 163.128.191.255/19

Bước 6: Dải khả dụng: 163.128.160.1 → 163.128.191.254

📌 Ví dụ 6: Chia subnet

Cho mạng 139.12.0.0/16 (lớp B), chia thành 4 subnet.

Bước 1: Y = 4 = 2^X → X = 2 bit mượn. Subnet mask mới = 16 + 2 = 18 bit → /18

Bước 2: Số host/subnet = 2^(32-18) - 2 = 2^14 - 2 = 16.382 host

Bước 3: SM = 255.255.192.0

Bước 4: Bước nhảy = 256 - 192 = 64 (ở octet 3)

4.11. Tóm tắt & Lời khuyên làm bài

💡 Quy trình 5 bước khi gặp bài subnetting
  1. Đọc prefix → biết số bit Network. Tính số bit Host = 32 - prefix.
  2. Xác định octet mượn (octet cuối cùng có bit = 1 trong mask) và số bit host trong octet đó.
  3. Đổi octet mượn của IP sang nhị phân, quan sát các bit host:
    • Tất cả = 0 → IP đã cho là Network
    • Tất cả = 1 → IP đã cho là Broadcast
    • Khác → IP đã cho là Host
  4. Tính Network: bit host = 0. Tính Broadcast: bit host = 1.
  5. Dải khả dụng: từ Network+1 đến Broadcast-1.
⚠️ Lỗi thường gặp
  • Quên trừ 2 khi tính số host (Network + Broadcast không gán được).
  • Nhầm octet mượn khi prefix không chia hết cho 8 (vd: /19, /26).
  • Sai khi đổi nhị phân sang thập phân (nhớ: 128-64-32-16-8-4-2-1).
  • Quên rằng IP 127.x.x.x là loopback (không gán được).
