Quản trị Windows Server

Chương này giới thiệu cách quản trị mạng trong môi trường Windows: từ User/Group, Active Directory, Group Policy đến phân quyền NTFS và giới hạn dung lượng Quota.

5.1. Hệ điều hành mạng (NOS)

NOS (Network Operating System) là hệ điều hành mạng — ngoài chức năng HĐH bình thường (như Windows 95/98), còn cung cấp dịch vụ mạng: chia sẻ file/máy in, quản lý tài khoản người dùng...

Một mạng yêu cầu 2 loại phần mềm:

  • HĐH máy trạm: Nhận yêu cầu từ người dùng. Nếu yêu cầu xử lý được cục bộ thì xử lý; không thì gửi cho server.
  • HĐH máy chủ: Lưu trữ phần lớn dữ liệu, thực hiện: quản lý tài khoản, central licensing, bảo vệ an ninh mạng & dữ liệu.

5.1.1. Đặc tính kỹ thuật của NOS

  • Multitasking (đa nhiệm): Thực thi nhiều nhiệm vụ cùng lúc, 1 CPU.
  • Multiprocessing (đa xử lý): Dùng nhiều CPU để xử lý 1 hoặc nhiều tiến trình.
  • Multiuser (đa người dùng): Nhiều người cùng truy cập cùng lúc.

5.1.2. Các NOS phổ biến

  • UNIX: Đa nhiệm, đa người, phục vụ mạng lớn. Nhược: nhiều version khác nhau, phức tạp, đòi hỏi máy mạnh.
  • Novell Netware: Mạng nhỏ 5-25 máy, nhỏ gọn, dễ cài. Phiên bản: 2.2, 3.11, 4.0, 4.1.
  • Windows: 2 loại:
    • WfC (Windows for Client): Mạng ngang hàng, <10 người, dùng chung ổ đĩa/máy in, không chạy chung ứng dụng.
    • WfS (Windows for Server): Công ty vừa/lớn, quản lý/bảo mật, dịch vụ phức tạp. Phiên bản: NT, 2000, 2003, 2008, 2012, 2016, 2019, 2022.

5.2. User Account & Group

Mỗi lần dùng Windows cần có User Account gồm Username + Password. Hệ thống phân biệt người này với người khác, từ đó cấp quyền truy cập tài nguyên.

5.2.1. Chứng thực người dùng

Khi user nhấn CTRL+ALT+DEL để logon:

  1. Hệ thống xác minh tài khoản trong file SAM (Security Accounts Manager).
  2. Nếu đúng → cấp quyền truy cập.

5.2.2. Local User Accounts

Lưu thông tin tài khoản cục bộ trên từng máy. Mặc định Windows tạo sẵn 2 account:

  • Administrator: Toàn quyền trên máy, quản lý tài nguyên và người dùng.
  • Guest: Truy xuất tài nguyên tạm thời. Mặc định bị Disable.

5.2.3. Group Account

Group là đối tượng đại diện cho 1 nhóm user, giúp dễ quản lý và cấp quyền. Các group mặc định trong Windows:

5.2.4. Nguyên tắc đặt tên tài khoản

  • Tên tài khoản phải duy nhất.
  • Dài tối đa 20 ký tự, không phân biệt hoa thường.
  • Không dùng ký tự đặc biệt (/ \ [ ] : ; | = , + * ? < >).
  • Không nên có khoảng trắng.
  • Không nên để trống password.

5.2.5. Tạo User / Group

Vào Computer Management → Local Users and Groups:

  • Tạo User: Right click User → New User → Nhập thông tin. Các tuỳ chọn:
    • User must change password at next logon: Buộc đổi pass lần đăng nhập đầu.
    • User cannot change password: Không cho user tự đổi pass.
    • Password never expires: Pass không hết hạn.
    • Account is disabled: Vô hiệu hoá account.
  • Tạo Group: Right click Groups → New Group → Nhập tên, có thể Add member ngay.

5.2.6. Group lồng Group

1 User có thể thuộc nhiều Group. 1 Group cũng có thể chứa nhiều Group con (lồng nhau). Ví dụ: Group "Học viên" chứa "Gr_Mạng" và "Gr_Visual", mỗi group con lại chứa User.

5.4. Active Directory (AD)

Active Directory là dịch vụ thư mục của Microsoft, lưu trữ thông tin về tài nguyên mạng (File Server, Printer, Fax, Application, Data, User, Group, Web Server) và cho phép tra cứu/sử dụng các tài nguyên này.

5.4.1. Chức năng của AD

  • Lưu giữ danh sách tập trung tài khoản người dùng, mật khẩu, tài khoản máy tính.
  • Cung cấp Domain Controller (DC) — server chứng thực/logon.
  • Duy trì bảng chỉ mục (Index) giúp máy dò tìm nhanh tài nguyên.
  • Tạo tài khoản với quyền khác nhau.
  • Chia nhỏ domain thành Subdomain hoặc OU, uỷ quyền cho admin bộ phận.

5.4.2. Lợi ích của AD

  • Centralized Management: Quản lý tập trung
  • Delegated Administration: Uỷ quyền quản trị
  • DNS Integration: Tích hợp DNS
  • Scalability: Mở rộng tốt (lên tới 10 triệu object/domain)

5.4.3. Các thành phần trong Directory Services

  • Object (đối tượng): Máy in, người dùng, server, trạm, thư mục dùng chung, dịch vụ mạng. Là thành tố cơ bản nhất.
  • Attribute (thuộc tính): Mô tả đối tượng (vd: mật khẩu, tên, IP).
  • Schema: Định nghĩa danh sách thuộc tính cho mỗi loại đối tượng. Có thể tuỳ biến.
  • Container (vật chứa): Tương tự thư mục Windows. 3 loại:
    • Domain: Đơn vị nòng cốt của AD.
    • Site: Phân biệt vị trí cục bộ và vị trí xa.
    • OU (Organizational Unit): Vật chứa người dùng, nhóm, máy tính, máy in, OU con.
  • Global Catalog: Xác định vị trí đối tượng người dùng được truy cập. Tìm kiếm bằng tên và thuộc tính.

5.4.4. Domain

Domain là đơn vị chức năng nòng cốt của AD. Là tập hợp người dùng, máy tính, tài nguyên chia sẻ quy tắc bảo mật giống nhau. Đáp ứng 3 chức năng:

  1. Common security policy: Chính sách bảo mật chung.
  2. Khu vực quản trị các đối tượng có chung CSDL, thư mục, chính sách bảo mật, quan hệ uỷ quyền.
  3. Domain Controller: Quản lý bảo mật tài nguyên, cung cấp server dự phòng, đồng bộ dữ liệu giữa các DC.

5.4.5. Domain Tree & Forest

  • Domain Tree: Nhiều domain sắp xếp theo cấu trúc cây. Domain đầu = Root, các domain sau = Child Domain. Tên phải khác biệt. VD: thbk.com → pm1.thbk.com, pm2.thbk.com.
  • Forest (Rừng): Tập hợp các Domain Tree có quan hệ uỷ quyền cho nhau. VD: dhcn.com + dhbk.com + các subdomain.

5.5. DCPROMO — Cài đặt Active Directory

Mặc định Windows Server mới cài là Standalone Server. DCPROMO (Active Directory Installation Wizard) dùng để:

  • Nâng cấp Standalone Server → Domain Controller (DC)
  • Hoặc giáng cấp DC → Standalone Server

5.5.1. Yêu cầu trước khi nâng cấp

  • Khai báo đầy đủ TCP/IP, đặc biệt DNS Server = chính IP của server nâng cấp.
  • Nên cài DNS trước, hoặc chọn cài tự động trong quá trình nâng cấp.

5.5.2. Các bước cài đặt

  1. Start → Run → dcpromo → OK
  2. Hộp cảnh báo: DOS, Win95, WinNT SP3 trở về trước sẽ bị loại khỏi AD → Next
  3. Chọn "Domain in a new forest" (nếu tạo domain đầu tiên)
  4. Nhập tên DNS đầy đủ của domain (vd: netclass.local)
  5. Nhập tên NetBIOS (mặc định)
  6. Chỉ định vị trí lưu database AD + log file
  7. Chỉ định thư mục SYSVOL (bắt buộc trên partition NTFS)
  8. Nếu DNS chưa cài → chọn "Install and configure DNS"
  9. Chọn mức tương thích server (2000/2003)
  10. Nhập mật khẩu Directory Services Restore Mode
  11. Xác nhận tóm tắt → Next → cài đặt → Finish → Reboot

5.5.3. Gia nhập máy trạm vào Domain

Sau khi có DC, máy trạm gia nhập domain:

  1. Đăng nhập máy cục bộ với Administrator.
  2. Right click My Computer → Properties → tab Computer Name → Change.
  3. Nhập tên domain ở "Member of Domain".
  4. Máy trạm tìm DC gần nhất → DC yêu cầu xác thực bằng tài khoản admin cấp miền.
  5. Sau khi thành công → reboot → hộp logon có 2 lựa chọn: logon vào domain hoặc máy cục bộ.

5.5.4. Organizational Unit (OU)

OU là nhóm người dùng, máy tính, tài nguyên mạng nhằm dễ quản lý và uỷ quyền cho admin địa phương. Đặc biệt: qua OU có thể áp đặt giới hạn phần mềm/phần cứng thông qua Group Policy.

Các bước xây dựng OU:

  1. Tạo OU: Right click domain → New → Organizational Unit → Nhập tên.
  2. Đưa máy trạm vào OU.
  3. Đưa tài khoản người dùng vào OU.
  4. Right click OU → Properties → tab Manager By → Change → chọn người quản lý OU.
  5. Tab Group Policy → New → Edit để tạo GPO cho OU.

5.6. Group Policy (Chính sách nhóm)

Group Policy là công cụ tích hợp sẵn (từ Windows XP) trong Microsoft Management Console. Chỉ thành viên Administrators mới có quyền dùng. Đây là nơi cấu hình chính sách cho toàn bộ thành phần máy: tài khoản, thiết bị, chương trình, bảo mật.

5.6.1. Khởi động

Start → Run → gpedit.msc

5.6.2. Hai nhánh chính

  • Computer Configuration: Áp dụng cho toàn bộ người dùng trên máy, bất kể ai đăng nhập.
    • Windows Settings: Chính sách tài khoản, password, khởi động, đăng nhập.
    • Administrative Templates:
      • Windows Components: IE, NetMeeting...
      • System: Tùy chọn chính sách hệ thống.
  • User Configuration: Áp dụng cho tài khoản đang sử dụng. Mặc định = "Not configured".

5.6.3. Cách sử dụng

Mở đến thành phần cần cấu hình → Properties → tab Setting. Có 3 trạng thái:

  • Not configured: Không định cấu hình.
  • Enabled: Kích hoạt.
  • Disabled: Vô hiệu hoá.

5.6.4. Account Policy (Chính sách tài khoản)

Áp dụng khi tiến trình Logon xảy ra. Cấu hình bảo mật cho mật khẩu, khoá tài khoản, chứng thực.

Account Lockout Policy

Quy định cách thức khoá tài khoản, hạn chế tấn công Logon từ xa.

  • Account Lockout Threshold: Số lần đăng nhập sai → khoá (vd: 5).
  • Account Lockout Duration: Thời gian khoá (mặc định 30 phút nếu Threshold được set).
  • Reset Lockout Counter After: Thời gian đếm lại số lần sai (mặc định 30 phút).

5.6.5. Audit Policy (Chính sách kiểm toán)

Ghi nhận sự kiện: đăng nhập, quản trị tài khoản, truy cập dịch vụ thư mục, logon/logoff, truy cập file/folder/printer, thay đổi chính sách, thay đổi quyền, hoạt động process, khởi động/tắt máy.

5.6.6. User Rights Assignment

2 cách cấp quyền hệ thống:

  1. Đưa User Account vào nhóm built-in (đã học).
  2. Dùng công cụ User Rights Assignment để gán từng quyền rời rạc.

Một số quyền:

  • Access this computer in the network
  • Add workstations to the domain
  • Backup files and directories
  • Change the system time
  • Force shutdown from a remote system
  • Deny logon locally / as service / as batch

5.6.7. Security Options

Một số lựa chọn bảo mật thông dụng:

  • Thay đổi trạng thái hoạt động của User (Enable/Disable)
  • Giới hạn user dùng password rỗng
  • Đổi tên tài khoản Administrator
  • Gửi message cho user khi logon
  • Không hiển thị tên user vừa logon
  • Cho phép Shutdown mà không cần logon
  • Không cần nhấn CTRL+ALT+DEL khi khởi động
  • Đổi tên tài khoản Guest

5.7. NTFS Permission

Từ Windows XP Professional, Microsoft bổ sung tính năng: bảo mật cấp độ folder/file, cấp hạn nghạch cho user, nén folder/file, mã hoá folder/file. Các tính năng này chỉ có trên partition định dạng NTFS.

Chuyển đổi FAT/FAT32 → NTFS mà không mất dữ liệu:

C:\> convert <ổ đĩa>: /fs:ntfs

5.7.2. Nguyên tắc thừa kế

  • Mặc định thư mục/file kế thừa quyền từ partition gốc → mọi Local User có 3 quyền tối thiểu.
  • Để cấu hình bảo mật riêng, phải tắt tính năng thừa kế: Properties → Security → Advanced → bỏ check "Inherit from parent...".
  • Check "Replace permission entries on..." → áp dụng cho cả thư mục con và file bên trong.

5.7.3. Cấp quyền truy cập

  1. Right click thư mục → Properties → tab Security → Add.
  2. Nhấn Advanced → Find Now → chọn user/group → OK.
  3. Mặc định user/group mới được 3 quyền tối thiểu. Có thể cấp thêm.
  4. Apply → OK.
ℹ️ NTFS vs Share Permission

Share Permission: Chỉ áp dụng khi truy cập qua mạng. NTFS Permission: Áp dụng cả khi truy cập mạng VÀ đăng nhập trực tiếp (local). NTFS chi tiết và bảo mật hơn.

5.8. Disk Quota (Giới hạn dung lượng)

Disk Quota giúp admin cấp dung lượng ổ cứng tối đa cho từng user cục bộ, tránh lãng phí. Quota chỉ set cho từng partition (không set cho cả ổ cứng).

5.8.1. Cấu hình

  1. Properties partition → tab Quota.
  2. Check "Enable quota management" → bật quản lý Quota.
  3. Check "Deny disk space to users exceeding quota limit" → chặn user dùng vượt hạn nghạch.
  4. Nhấn Quota Entries → Quota → New Quota Entry → Add user → nhập dung lượng → OK.
⚠️ Lưu ý

Nếu chỉ bật "Enable quota management" mà KHÔNG check "Deny disk space to users exceeding quota limit" → user vẫn có thể ghi vượt dung lượng được cấp (chỉ bị cảnh báo). Phải check cả 2 ô mới chặn được.

5.9. Nén & Mã hoá thư mục

5.9.1. Nén thư mục (Compression)

  1. Right click thư mục → Properties → Advanced.
  2. Check "Compress contents to save disk space" → OK → Apply.
  3. Chọn kiểu nén → OK.

Thư mục nén có icon chữ màu xanh dương. Trình nén hoạt động ngầm — user vẫn mở/đọc/lưu file bình thường.

5.9.2. Mã hoá thư mục (Encryption - EFS)

  1. Right click thư mục → Properties → Advanced.
  2. Check "Encrypt contents to secure data" → OK → Apply.

Thư mục mã hoá có icon màu xanh lá. Đặc điểm:

  • Tiến trình mã hoá hoàn toàn tự động.
  • Chỉ tài khoản đã mã hoá mới làm việc được với thư mục đó.
  • Khi hệ điều hành lỗi hoặc đem file sang PC khác → không sử dụng được.

5.10. Tóm tắt chương

  • NOS = HĐH mạng, đa nhiệm/đa xử lý/đa người. Phổ biến: Windows Server, UNIX, Netware.
  • User Account: Username + Password, lưu trong SAM (cục bộ) hoặc NTDS.DIT (domain).
  • Group: Nhóm user, dễ quản lý. Administrators có toàn quyền, Users dùng cơ bản.
  • Workgroup = Peer-to-Peer, SAM cục bộ, <15 máy. Domain = Client-Server, AD tập trung.
  • Active Directory: Domain → OU → Tree → Forest. Lưu trong NTDS.DIT trên Domain Controller.
  • DCPROMO: Nâng cấp Server → DC. Cần khai báo DNS.
  • Group Policy (gpedit.msc): Computer Config + User Config. Quản lý Password, Lockout, Audit, Rights, Security.
  • NTFS Permission: 13 quyền chi tiết, có thừa kế. FAT→NTFS bằng lệnh convert.
  • Disk Quota: Giới hạn dung lượng từng user. Phải check "Deny disk space..." để chặn thật.
  • Nén (icon xanh dương) & Mã hoá EFS (icon xanh lá): chỉ có trên NTFS.
