Mô hình OSI & TCP/IP và các giao thức

Để các máy tính khác hãng, khác hệ điều hành vẫn "nói chuyện" được với nhau, người ta chuẩn hóa kiến trúc mạng theo mô hình phân tầng. Hai mô hình quan trọng nhất là OSI (7 tầng, lý thuyết) và TCP/IP (4 tầng, thực tế).

3.1. Giao thức (Protocol) là gì?

Giao thức là tập hợp các nguyên tắc, quy định về truyền nhận thông tin giữa các máy tính và thiết bị trên mạng — bao gồm cấu trúc dữ liệu và phương thức truyền.

Một số bộ giao thức lịch sử:

  • NetBEUI: IBM + Microsoft thiết kế cho LAN nhỏ/cỡ vừa. Không hỗ trợ định tuyến.
  • IPX/SPX: Dùng trong mạng Novell Netware. Nhỏ, nhanh, có hỗ trợ định tuyến.
  • TCP/IP: Chuẩn toàn cầu của Internet hiện nay.
🌐 Ví dụ đời thường

Giao thức giống như ngôn ngữ giao tiếp. Nếu 2 người đều nói tiếng Anh thì nói chuyện được. Nếu 1 nói Việt, 1 nói Trung thì cần "Gateway" phiên dịch. TCP/IP như tiếng Anh quốc tế — ai cũng dùng.

3.2. Tại sao phải phân tầng?

  • Giảm độ phức tạp: Chia bài toán lớn thành các phần nhỏ, mỗi tầng lo 1 việc.
  • Chuẩn hóa: Các hãng khác nhau làm theo cùng chuẩn → tương thích.
  • Modun hóa: Có thể thay đổi 1 tầng mà không ảnh hưởng tầng khác (vd: đổi cáp đồng sang quang mà ứng dụng web vẫn chạy bình thường).
  • Dễ bảo trì, sửa lỗi: Lỗi ở tầng nào tìm tầng đó, không phải kiểm tra toàn hệ thống.
  • Thúc đẩy tiến hóa: Phát triển từng tầng độc lập.

3.3. Mô hình OSI (7 tầng)

OSI (Open Systems Interconnection) ra đời năm 1984, là mô hình tham chiếu lý thuyết gồm 7 tầng. Tầng thấp nhất (1) định nghĩa giao tiếp vật lý, tầng cao nhất (7) định nghĩa ứng dụng.

3.3.1. Chi tiết từng tầng

Tầng 7 — Application (Ứng dụng)

Cung cấp giao tiếp trực tiếp giữa chương trình ứng dụng và hệ thống mạng. Ví dụ: trình duyệt web dùng HTTP, FTP client dùng FTP, mail client dùng SMTP/POP3/IMAP. Cung cấp cơ chế xác thực người dùng.

Tầng 6 — Presentation (Trình diễn)

Đảm bảo bên nhận hiểu được dữ liệu bên gửi. Chuyển đổi cú pháp, định dạng (ASCII, EBCDIC, JPEG, GIF, MPEG), mã hóa/giải mã bảo mật (SSL/TLS), nén/giải nén dữ liệu để tiết kiệm băng thông.

Tầng 5 — Session (Phiên)

Thiết lập, duy trì, đồng bộ và kết thúc các "phiên" (session) giao dịch giữa ứng dụng 2 máy. Có cơ chế checkpoint để phục hồi nếu phiên bị gián đoạn.

Tầng 4 — Transport (Giao vận)

Đảm bảo truyền tải đầu cuối (End-to-End) — từ máy nguồn đến máy đích. Cắt dữ liệu thành các Segment, đánh số thứ tự, kiểm soát luồng, phát hiện và sửa lỗi. Có 2 giao thức chính: TCP (tin cậy) và UDP (nhanh nhưng không tin cậy).

Tầng 3 — Network (Mạng)

Định tuyến gói tin, chọn đường đi tốt nhất. Cấp địa chỉ logic IP cho thiết bị. Đơn vị dữ liệu: Packet (Gói tin). Router hoạt động ở tầng này.

Tầng 2 — Data Link (Liên kết dữ liệu)

Đóng dữ liệu thành Frame, gắn địa chỉ MAC nguồn & đích. Kiểm soát truy nhập môi trường truyền (ai được gửi khi), phát hiện lỗi qua trailer (FCS). Switch, Bridge hoạt động ở tầng này.

Tầng 1 — Physical (Vật lý)

Chuyển đổi dữ liệu thành các bit nhị phân (0, 1) và phát các xung tín hiệu điện/quang/sóng lên dây cáp. Quy định loại cáp, mức điện áp, tốc độ bit, đầu nối vật lý. Repeater, Hub hoạt động ở tầng này.

3.4. Quá trình Đóng gói (Encapsulation) & Gỡ gói (Decapsulation)

Khi dữ liệu đi từ tầng 7 xuống tầng 1 ở máy gửi, mỗi tầng thêm 1 header (và tầng 2 thêm cả trailer) — gọi là đóng gói (Encapsulation). Khi đến máy nhận, dữ liệu đi từ tầng 1 lên tầng 7, mỗi tầng gỡ bỏ 1 header — gọi là gỡ gói (Decapsulation).

4Segment (TCP) / Datagram (UDP)Transport Header (port, seq, ack)

🏠 Ví dụ đời thường

Bạn gửi 1 bức thư: thư (data) → phong bì (transport header) → ghi địa chỉ người nhận (network header IP) → mã bưu chính (MAC) → bưu tá đưa (bit vật lý). Mỗi lớp thêm 1 thông tin để định tuyến.

3.5. Mô hình TCP/IP (4 tầng)

Là mô hình thực tế vận hành Internet ngày nay, rút gọn từ OSI.

  • Application: Gộp tầng 5,6,7 OSI. (HTTP, FTP, SMTP, DNS, DHCP...)
  • Transport: Tương đương tầng 4 OSI. (TCP, UDP)
  • Internet: Tương đương tầng 3 OSI. (IP, ICMP, ARP, IGMP)
  • Network Access: Gộp tầng 1,2 OSI. (Ethernet, Wi-Fi, PPP)

3.7. TCP vs UDP — Hai giao thức tầng Transport

TCP (Transmission Control Protocol)

Hướng kết nối, tin cậy cao.

  • Thiết lập kết nối trước bằng 3-way handshake (SYN → SYN-ACK → ACK).
  • Đánh số thứ tự (Sequence number) các gói tin.
  • Yêu cầu xác nhận (ACK) — mất gói thì gửi lại.
  • Điều khiển luồng, kiểm soát tắc nghẽn.
  • Ứng dụng: Web (HTTP), Email (SMTP), FTP, SSH — cần chính xác.
UDP (User Datagram Protocol)

Không kết nối, không tin cậy, nhưng cực nhanh.

  • Không thiết lập kết nối — gửi ngay.
  • Không đánh số thứ tự, không ACK, không gửi lại.
  • Tiết kiệm overhead, độ trễ thấp.
  • Ứng dụng: Streaming video, Voice/Video call, Game online, DNS — cần tốc độ, chấp nhận mất gói.
🏠 Ví dụ đời thường

TCP như gửi hàng bằng đường bưu điện có EMS: ký nhận, nếu mất thì gửi lại, chậm nhưng chắc. UDP như phát sóng radio: đài cứ phát, ai bắt được thì nghe, mất 1 đoạn thì cũng kệ — quan trọng là nghe tức thời.

3.9. DHCP (Dynamic Host Configuration Protocol)

DHCP là giao thức cấp phát địa chỉ IP động cho thiết bị khi tham gia mạng. Hoạt động theo mô hình Client-Server.

3.9.1. Ưu điểm DHCP

  • Quản lý mạng dễ dàng, không cần cấu hình thủ công từng máy.
  • Tránh trùng lặp IP.
  • Giảm IP không sử dụng (thu hồi khi hết hạn).
  • Thiết bị tự cấu hình khi kết nối → tiện lợi.

3.9.2. Quy trình DORA (4 bước)

  1. Discover: Client broadcast "Tôi cần IP!" (DHCPDISCOVER)
  2. Offer: DHCP Server đề xuất 1 IP cho client (DHCPOFFER)
  3. Request: Client đồng ý và xin cấp IP đó (DHCPREQUEST)
  4. Acknowledge: Server xác nhận, cấp IP + Subnet Mask + Gateway + DNS (DHCPACK)
ℹ️ Tĩnh vs Động

IP tĩnh: Cố định, do người cấu hình thủ công — không đổi đến khi cấu hình lại. Phù hợp Server, Printer, Router.
IP động: Do DHCP Server cấp, có thời hạn (lease time), hết hạn thì xin lại hoặc cấp IP khác. Phù hợp PC user, mobile.

3.9.3. Lệnh kiểm tra trên Windows

# Xem cấu hình IP hiện tại
C:\> ipconfig

# Xem chi tiết kèm MAC
C:\> ipconfig /all

# Trả IP về DHCP Server
C:\> ipconfig /release

# Xin cấp lại IP
C:\> ipconfig /renew

3.9.4. ip helper-address

Khi DHCP Client và DHCP Server khác mạng (qua Router), gói tin DHCPDISCOVER (broadcast) sẽ bị Router chặn. Để giải quyết, dùng lệnh trên cổng Router: 

Router(config)# interface <tên cổng>
Router(config-if)# ip helper-address <địa chỉ IP DHCP Server>

Lệnh này chuyển tiếp gói DHCP Broadcast thành Unicast tới DHCP Server.

3.9.5. Vấn đề bảo mật DHCP

  • DHCP Client bất hợp pháp: Kẻ tấn công xin nhiều IP → cạn kiệt pool IP. Giải pháp: port security trên Switch.
  • Man-in-the-middle: Kẻ tấn công mạo danh DHCP Server cấp IP giả. Giải pháp: DHCP Snooping trên Switch.

3.10. DNS (Domain Name System)

DNS là hệ thống phân giải tên miền — dịch tên miền dễ nhớ (google.com) sang địa chỉ IP khó nhớ (142.250.74.14) để máy tính kết nối được.

3.10.1. Nguyên tắc hoạt động

  1. Nhập "google.com" vào trình duyệt.
  2. Máy hỏi DNS Server sơ cấp (thường là của ISP hoặc 8.8.8.8 của Google).
  3. Nếu DNS sơ cấp không biết → hỏi lên DNS cấp cao hơn (TLD .com, rồi DNS của google.com).
  4. DNS trả IP cho máy → máy kết nối tới web server của google.com.
  5. Máy lưu IP vào DNS cache để lần sau không phải hỏi lại.

3.10.2. Các loại DNS Record

  • A Record: Trỏ tên miền → IPv4
  • AAAA Record: Trỏ tên miền → IPv6
  • CNAME: Tên miền bí danh (alias) trỏ tới tên miền khác
  • MX Record: Mail Exchange — chỉ định mail server cho domain
  • TXT Record: Lưu thông tin văn bản (SPF, DKIM...)
  • NS Record: Name Server — chỉ định DNS server quản lý domain

3.10.3. Bảo mật DNS

  • Tấn công DoS/DDoS DNS: Làm quá tải DNS server → user không vào web được.
  • Giả mạo DNS (DNS Spoofing/Poisoning): Đổi record DNS → dẫn user tới web giả mạo.
  • DNS Leak: Khi dùng VPN mà truy vấn DNS không đi qua VPN → lộ lịch sử web.
  • Giải pháp: DNSSEC (chữ ký số), cập nhật vá lỗi, giới hạn truy vấn.

3.11. ARP & ICMP

3.11.1. ARP (Address Resolution Protocol)

ARP phân giải chiều ngược lại DNS: tìm địa chỉ MAC từ địa chỉ IP đã biết. Khi PC A muốn nói chuyện với PC B cùng mạng LAN, A chỉ biết IP của B → cần MAC để đóng frame.

  1. A gửi ARP Request broadcast: "Ai có IP 192.168.1.5, cho tôi xin MAC!"
  2. Tất cả máy trong mạng nhận request.
  3. B (có IP trùng) trả ARP Reply unicast: "Tôi đây, MAC là XX:XX:XX:XX:XX:XX"
  4. A lưu vào ARP Cache để lần sau không phải hỏi lại.

3.11.2. ICMP (Internet Control Message Protocol)

ICMP là giao thức báo lỗi hoạt động hợp nhất với IP. Các lệnh mạng phổ biến dùng ICMP:

  • ping: Gửi ICMP Echo Request → đích trả Echo Reply. Kiểm tra thông mạng và đo thời gian往返.
  • traceroute / tracert: Liệt kê danh sách các Router (hop) mà gói tin đi qua để tới đích.
# Ping 4 gói tới google
C:\> ping google.com

# Trace đường đi tới google (Windows)
C:\> tracert google.com

# Trace (Linux/Mac)
$ traceroute google.com

3.12. Tóm tắt chương

  • Giao thức = quy tắc truyền nhận dữ liệu. TCP/IP là chuẩn toàn cầu.
  • Phân tầng để giảm phức tạp, chuẩn hóa, dễ bảo trì.
  • OSI 7 tầng: Application → Presentation → Session → Transport → Network → Data Link → Physical.
  • TCP/IP 4 tầng: Application (gộp 5,6,7) → Transport → Internet (tầng 3) → Network Access (gộp 1,2).
  • Đóng gói (xuống): Data → Segment → Packet → Frame → Bit. Gỡ gói (lên): ngược lại.
  • TCP = tin cậy (3-way handshake, ACK). UDP = nhanh (streaming, game).
  • DHCP cấp IP động qua DORA. DNS phân giải tên miền → IP.
  • ARP tìm MAC từ IP. ICMP báo lỗi (ping, traceroute).
