Cloud Service Models · IaaS / PaaS / SaaS

Click vào từng mô hình để xem phân chia trách nhiệm giữa nhà cung cấp và khách hàng. Càng lên cao (SaaS), nhà cung cấp lo càng nhiều — khách hàng càng nhàn.

IaaS
Infrastructure as a Service
PaaS
Platform as a Service
SaaS
Software as a Service
💡 Quy tắc nhớ

Càng lên cao (SaaS), nhà cung cấp chịu trách nhiệm càng nhiều:

  • IaaS: Khách hàng cài HĐH, Database, Code — lo nhiều nhất
  • PaaS: Khách hàng chỉ lo Code — nhà cung cấp lo HĐH, DB, Runtime
  • SaaS: Khách hàng chỉ việc dùng — nhà cung cấp lo toàn bộ
🏠 Ví dụ đời thường

IaaS = thuê đất trống, tự xây nhà, tự nội thất.
PaaS = thuê căn hộ trống, chỉ việc mang đồ dùng vào.
SaaS = thuê khách sạn, vào ở ngay, xách vali đến.
