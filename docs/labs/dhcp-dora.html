DHCP DORA · Cấp phát IP động

Khi máy tính kết nối vào mạng, nó cần 1 IP. DHCP Server cấp IP động qua quy trình 4 bước D-O-R-A: Discover → Offer → Request → Acknowledge.

Client
💻 PC mới kết nối
IP: 0.0.0.0
Server
🖥️ DHCP Server
Pool: 192.168.1.10-100
💻 Client
broadcast
DHCPDISCOVER
(broadcast)
🖥️ Server
nhận được
① DISCOVER
Client mới vào mạng chưa có IP → gửi broadcast "Có DHCP Server nào không? Tôi cần IP!" (src: 0.0.0.0, dst: 255.255.255.255)
💻 Client
nhận offer
DHCPOFFER
(unicast)
🖥️ Server
đề xuất IP
② OFFER
DHCP Server đề xuất IP cho client: "Tôi có IP 192.168.1.50 cho bạn, kèm Subnet Mask, Gateway, DNS"
💻 Client
broadcast
DHCPREQUEST
(broadcast)
🖥️ Server
xin cấp IP đó
③ REQUEST
Client đồng ý nhận IP đó. Gửi broadcast "Tôi xin nhận IP 192.168.1.50 từ DHCP Server này" (để các DHCP Server khác rút offer)
💻 Client
nhận xác nhận
DHCPACK
(unicast)
🖥️ Server
xác nhận cấp
④ ACK
Server xác nhận cuối cùng: "OK, IP 192.168.1.50 là của bạn trong 8 giờ tới (lease time). Hết hạn xin lại!"
✓ Hoàn tất! Client đã nhận IP 192.168.1.50, Subnet Mask 255.255.255.0, Gateway 192.168.1.1, DNS 8.8.8.8.
ℹ️ Khi Client và DHCP Server khác mạng

DHCPDISCOVER là broadcast → bị Router chặn. Nếu Client ở mạng A, DHCP Server ở mạng B: cần cấu hình trên cổng Router của mạng A: 

Router(config)# interface <cổng mạng A>
Router(config-if)# ip helper-address <IP DHCP Server>

Lệnh này chuyển DHCP Broadcast thành Unicast gửi thẳng tới DHCP Server.

🏠 Ví dụ đời thường

DHCP giống bạn vào quán cafe, hỏi "Cho tôi 1 chỗ ngồi" (DISCOVER) → nhân viên "Còn bàn số 5, anh ngồi không?" (OFFER) → bạn "Vâng, tôi ngồi bàn 5" (REQUEST) → nhân viên "Vâng mời anh, bàn 5 là của anh trong 2 tiếng" (ACK).
